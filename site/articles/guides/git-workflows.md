# Git Workflows

## Reference selection

Use the Git tab to search local branches, remote branches, GitHub pull requests, and GitLab merge requests. The app prefers `upstream` when it exists, falls back to `origin`, and paginates provider API results.

## Review request state

Settings can show open, closed, merged, or all PRs/MRs. Open requests are the default to match normal review workflows.

## Open without replacing current work

Right-click a branch, PR, or MR to open it as a new workspace graph tab. This keeps the current graph loaded while another review context is prepared.

## Range comparisons

History context menus can copy a commit hash or set it as the comparison range start/end. This makes it easier to build custom range diffs without manual copy errors.
